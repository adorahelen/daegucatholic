추가 개선 사항

1. 테스트 데이터 확인:

테스트 실행 전 데이터가 초기 상태임을 보장하기 위해 기존 데이터를 확인하거나 초기화합니다.

2. 통합 테스트 전용 데이터 사용:

@Sql 어노테이션을 사용하여 테스트 데이터가 겹치지 않도록 보장할 수 있습니다. 예를 들어, 테스트 시작 전 초기 데이터를 삽입하거나 삭제하도록 SQL 스크립트를 실행합니다.
